# IMX v0.11.0 Progressive JPEG Input Slice

v0.11.0 keeps the v0.10.0 JPEG EXIF Orientation fixture surface and adds bounded input support for
ordinary 8-bit progressive JPEG photos.

## Contract

- `.jpg`, `.jpeg`, and exact `JPEG:` remain the only JPEG path/prefix forms.
- Progressive JPEG support is input-only for 8-bit grayscale and RGB streams.
- Baseline JPEG input remains supported.
- EXIF Orientation values 1 through 8 are still read before decode for both
  baseline and progressive input.
- Decoded pixels are normalized according to Orientation, and `identify`
  reports oriented dimensions.
- JPEG output remains deterministic fixed quality-90 baseline JPEG.
- Progressive scan layout, source bytes, EXIF, ICC, XMP, density, thumbnail,
  timestamp, GPS, camera, profile, and color-management metadata are not
  preserved or interpreted beyond read-only Orientation.

## Proof Requirements

- Codec tests cover progressive RGB, progressive grayscale, and progressive
  Orientation input.
- CLI tests prove exact `JPEG:` identify/transcode for progressive RGB,
  progressive grayscale, and progressive Orientation input.
- Malformed/truncation tests cover progressive JPEG truncation without panics.
- Generated fixtures include `progressive-rgb-4x3.jpg`,
  `progressive-gray-4x2.jpg`, and `progressive-orientation-o6.jpg`.
- Differential corpus compares progressive JPEG output against ImageMagick with
  existing JPEG lossy metric thresholds.
- Install, package, archive, and tap formula smoke checks include progressive
  JPEG input.

## Still Out Of Scope

CMYK/YCCK JPEG, 12-bit JPEG, arithmetic-coded JPEG, lossless JPEG/JPEG-LS, JPEG
2000, JPEG XL, progressive JPEG output, metadata preservation, EXIF writing,
GPS handling, ICC/profile transforms, color management, resize/crop/filter
transforms, full ImageMagick CLI parsing, MagickCore, and MagickWand remain out
of scope.
