# IMX v0.18.0 JSON Identify and Report

`v0.18.0` adds one bounded machine-readable metadata slice over the existing
identify surface.

## Supported Surface

- `imx identify --json [FORMAT:]<input>`.
- `imx report --json [FORMAT:]<input>`.
- Existing text `imx identify [FORMAT:]<input>` output remains unchanged.
- JSON covers BMP, FARBFELD, JPEG, QOI, PBM, PGM, PNG, and PPM through the
  existing extension and exact-prefix contract.

## JSON Contract

`imx identify --json` prints one JSON object on success:

```json
{"schema_version":1,"format":"PPM","width":2,"height":1,"channels":"RGB","depth":8}
```

`imx report --json` prints one JSON object with a status:

```json
{"schema_version":1,"status":"supported","diagnostic_code":null,"format":"PPM","width":2,"height":1,"channels":"RGB","depth":8}
```

On unsupported, unreadable, or malformed inputs, `report --json` exits `0` and
prints:

```json
{"schema_version":1,"status":"unsupported","diagnostic_code":"input.unsupported_format_prefix","message":"unsupported format prefix: GIF"}
```

`identify --json` prints the same diagnostic JSON to stderr and exits `1` for
data, IO, malformed-input, and validation failures. Invalid command shapes still
exit `2` with usage text.

The shared identify fields are the existing identify fields only. Width, height,
and depth are JSON numbers; format and channels are uppercase strings matching
the text identify contract. JPEG width and height are the existing
Orientation-normalized dimensions where EXIF Orientation applies.

## Diagnostic Codes

The stable v0.18 diagnostic-code surface is intentionally small:

- `input.unsupported_format_prefix`
- `input.missing_prefix_path`
- `input.missing`
- `input.read_failed`
- `input.too_large`
- `input.unsupported_format`
- `input.format_prefix_mismatch`
- `image.invalid_header`
- `image.unexpected_eof`
- `image.invalid_dimensions`
- `image.length_overflow`
- `image.too_large`
- `image.invalid_pixel_buffer`
- `image.allocation_failed`
- `image.unsupported_format`
- `qoi.invalid_channels`
- `qoi.invalid_colorspace`
- `pnm.invalid_max_value`
- `pnm.invalid_sample_value`
- `pbm.invalid_sample`
- `bmp.unsupported_feature`
- `jpeg.identify_failed`
- `jpeg.decode_failed`
- `png.identify_failed`
- `png.decode_failed`

## Non-Goals

No new image formats, no extra metadata extraction, no EXIF/ICC/XMP/profile
reporting beyond existing JPEG Orientation dimensions, no histograms, hashes,
pixel dumps, recursive walking, glob expansion, stdin/stdout streaming,
ImageMagick JSON schema compatibility, `magick` alias, full ImageMagick CLI
parsing, delegates, MagickCore, MagickWand, Homebrew/core, crates.io, Windows,
or hosted macOS/iOS Actions.

## Evidence

- CLI tests cover JSON identify across all supported formats, unprefixed and
  prefixed inputs, high-depth PPM, and JPEG Orientation dimensions.
- CLI tests cover `report --json` supported output, unsupported prefixes,
  missing prefixed paths, mismatched prefixes, missing inputs, malformed QOI,
  exact command-shape rejection, and JSON identify error output.
- `imx self-test`, installer smoke, source install verification, packaged
  archive smoke, published archive smoke, and generated Homebrew formula smoke
  include direct JSON identify/report checks.
- Generated conformance wording and summary record the JSON identify/report
  contract without adding broader metadata claims.
