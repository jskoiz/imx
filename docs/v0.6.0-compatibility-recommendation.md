# IMX v0.6.0 Compatibility Recommendation

Status: recommendation only. Do not implement this slice until accepted.

## Recommended Slice

Add ImageMagick-style format prefixes for existing supported formats only:

- `FARBFELD:input.ff`
- `QOI:input.qoi`
- `PBM:input.pbm`
- `PGM:input.pgm`
- `PPM:input.ppm`

The prefix should select or confirm the format for file paths that already map
to the existing FARBFELD/QOI/PBM/PGM/PPM codecs. It must not add stdin/stdout,
new formats, ImageMagick command parsing, delegates, transforms, MagickCore, or
MagickWand.

## Likely Files

- `crates/cli/src/main.rs`
- `crates/cli/tests/cli.rs`
- `scripts/differential-corpus.sh`
- `README.md`
- `COMPATIBILITY.md`
- `RELEASE_NOTES.md`
- `PRODUCTION_READINESS.md`

## Required Proof

- CLI tests for prefixed identify and transcode forms.
- Differential corpus cases that invoke IMX with prefixes and compare decoded
  pixels through the ImageMagick oracle.
- Malformed prefix tests for unknown prefixes, mismatched prefixes, missing
  paths, and same-path rejection.
- Existing release gates, fuzz smoke, benchmark smoke/regression, install smoke,
  package smoke, archive smoke, and no-ImageMagick-linkage checks.

## Non-Goals

- No PNG, JPEG, TIFF, PAM, PFM, BMP, or high-depth PPM.
- No full `magick` CLI parser or binary alias.
- No stdin/stdout streaming.
- No delegates, profiles, color management, resize/transform operations,
  MagickCore API, or MagickWand API.

## Rollback Risk

Low if prefixes are parsed as a CLI path adapter before existing decode/encode
dispatch. The main risk is silently accepting mismatched prefixes; that should
fail with a clear unsupported-format error instead of guessing.
