# IMX v0.7.0 High-Depth PPM Candidate

Status: implementation candidate in source. No v0.7.0 tag, GitHub release,
published archive, or tap update is claimed by this document.

## Objective

Make the existing PPM surface materially more useful without broadening IMX into
a new format family. v0.7.0 adds high-depth PPM handling for uppercase `P3` and
`P6` only, while preserving the v0.6.0 prefix and same-format rewrite contract.

## Supported Delta

- `imx identify input.ppm` and `imx identify PPM:input.ppm` report high-depth
  PPM as `format=PPM ... channels=RGB depth=16`.
- ASCII `P3` and binary `P6` accept `maxval` in `1..=65535`.
- `maxval <= 255` decodes as RGB8. `maxval > 255` decodes as RGB16BE.
- Binary `P6` high-depth samples are two big-endian bytes per channel.
- PPM output is deterministic binary `P6`.
- PPM output uses `maxval 255` for 8-bit sources and `maxval 65535` for 16-bit
  RGB/RGBA/GRAY sources.
- PPM16 to FARBFELD and PGM16 preserves 16-bit sample precision.
- PPM16 to QOI or PBM is intentionally lossy because those destinations are
  8-bit or bilevel.

## Preserved v0.6.0 Contract

- Exact uppercase prefixes remain limited to `FARBFELD:`, `QOI:`, `PBM:`,
  `PGM:`, and `PPM:`.
- Prefixes only confirm selected input/output format and do not add
  extensionless output selection.
- Same-format rewrites still require different paths and remain deterministic
  decode/re-encode operations.
- The product binary remains `imx`, not `magick`.

## Non-Goals

- No PNG, JPEG, TIFF, PAM, PFM, BMP, or additional file formats.
- No stdin/stdout streaming.
- No `magick` alias or full ImageMagick CLI parser.
- No delegates, profiles, transforms, color management, MagickCore, or
  MagickWand.
- No crates.io, Homebrew/core, Windows, tag, GitHub release, or tap update in
  this implementation slice.
- No hosted macOS or iOS GitHub Actions.

## Required Local Evidence

The candidate is ready for a future release goal only after these local gates
pass from the committed source state:

```sh
cargo fmt --all -- --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
IMAGEMAGICK_MAGICK=target/local-tools/magick-oracle IMX_REQUIRE_ORACLE=1 cargo test --test differential -- --nocapture
IMAGEMAGICK_MAGICK=target/local-tools/magick-oracle bash scripts/differential-corpus.sh
IMX_FUZZ_MAX_TOTAL_TIME=5 bash scripts/run-fuzz.sh
IMAGEMAGICK_MAGICK=target/local-tools/magick-oracle IMX_BENCH_ITERATIONS=3 bash scripts/bench-release.sh
IMAGEMAGICK_MAGICK=target/local-tools/magick-oracle IMX_BENCH_BASE_REF=v0.5.0 IMX_BENCH_ITERATIONS=3 bash scripts/bench-regression.sh
IMX_INSTALL_REPO_URL=/Users/jk/Desktop/imx IMX_INSTALL_REVISION=HEAD bash scripts/verify-install.sh
bash scripts/package-release.sh
```

Archive smoke should be run against the package target produced by
`scripts/package-release.sh` before publication claims are made:

```sh
IMX_RELEASE_DIR=target/release-artifacts \
  IMX_RELEASE_TARGET="$(rustc -vV | sed -n 's/^host: //p')" \
  bash scripts/verify-release-archive.sh
```

## Publication Gate

Publishing v0.7.0 is a separate goal. A release/tap goal must verify GitHub
release assets from published `SHA256SUMS`, update the tap from those published
checksums, and keep hosted proof Ubuntu/Linux-only.
