# IMX v0.6.0 Release-Ready Checklist

Status: post-release production-candidate evidence record. The `v0.6.0` tag
points at the approved release commit, GitHub Release assets are verified from
the published `SHA256SUMS`, and the Homebrew tap update is generated from those
published checksums before support is claimed.

## Release Surface

- Binary: `imx`.
- Formats: FARBFELD, QOI, PBM, PGM, PPM.
- Commands: `imx identify [FORMAT:]<input>` and
  `imx [FORMAT:]<input> [FORMAT:]<output>`.
- Compatibility delta: exact uppercase `FARBFELD:`, `QOI:`, `PBM:`, `PGM:`,
  and `PPM:` prefixes for existing identify and two-path transcode operands.
- Carried-forward behavior: deterministic same-format rewrites for the five
  supported formats when input and output paths differ.
- Hosted release targets: `x86_64-unknown-linux-gnu` and
  `aarch64-unknown-linux-gnu`.

## Publication Gate

Before pushing, tagging, or dispatching any workflow, rerun the hosted Apple
Actions guard and inspect workflow runners. Publication is allowed only if the
workflows remain Ubuntu/Linux-only and the working tree is intentional.

## Post-Publish Evidence

After tag publication, verify the published Linux x86_64 and Linux arm64
archives from GitHub Release assets, confirm `SHA256SUMS`, `imx.rb`,
`CONFORMANCE_REPORT.md`, and `conformance-summary.json`, then update
`jskoiz/homebrew-imx` from the published `SHA256SUMS` and prove Linux tap smoke.

## Non-Goals

No PNG, JPEG, TIFF, PAM, PFM, BMP, high-depth PPM, stdin/stdout, full
ImageMagick CLI parsing, `magick` alias, delegates, profiles, transforms,
MagickCore, MagickWand, upstream ImageMagick work, Homebrew/core, crates.io,
Windows, or hosted macOS/iOS Actions.
