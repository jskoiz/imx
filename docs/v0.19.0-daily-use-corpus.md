# IMX v0.19.0 Daily-Use Corpus Hardening

`v0.19.0` is a production-confidence milestone for the existing supported
surface. It does not add formats, commands, metadata extraction, or broader
ImageMagick CLI behavior.

## Supported Surface

- Existing BMP, FARBFELD, JPEG, QOI, PBM, PGM, PNG, and PPM support.
- Existing `identify`, `identify --json`, `report --json`, two-path transcode,
  `resize`, `resize-fit`, `batch-convert`, and `self-test` commands.
- Existing exact uppercase `BMP:`, `FARBFELD:`, `JPEG:`, `QOI:`, `PBM:`,
  `PGM:`, `PNG:`, and `PPM:` prefixes.

## Daily-Use Corpus Contract

`scripts/daily-use-corpus.sh` is a no-oracle installed-binary confidence gate.
It uses the generated fixture corpus and an actual `imx` binary to prove:

- `identify --json` and `report --json` for generated fixtures in every
  supported format family.
- Additional representative intake variants for top-down/RGBA BMP,
  progressive camera-style JPEG Orientation, QOI RGB linear, PGM/PPM
  comment/CRLF cases, and RGB16 PNG.
- Representative prefixed transcodes across all supported format families.
- Stable `report --json` unsupported diagnostics for unsupported prefixes,
  missing prefix paths, missing inputs, prefix mismatches, unsupported formats,
  malformed QOI, malformed PNM max values, and unsupported BMP compression.
- `identify --json` failure JSON on stderr for malformed and mismatched input.

The script writes:

- `target/daily-use-corpus/manifest.json`
- `target/daily-use-corpus/results.jsonl`
- `target/daily-use-corpus/summary.json`

## Release/Install Gates

The daily-use corpus gate is run by:

- `scripts/ci.sh`
- `scripts/verify-install.sh`
- `scripts/package-release.sh`
- `scripts/verify-release-archive.sh`
- generated conformance evidence lookup

Homebrew formula generation for v0.19.0 and newer also carries direct
diagnostic JSON smoke checks for unsupported prefixes and prefix mismatches.

## Non-Goals

- No new image formats.
- No stdin/stdout, recursive reporting, recursive batch mode, or watch mode.
- No full ImageMagick CLI parser or `magick` alias.
- No ImageMagick JSON schema compatibility.
- No metadata/profile/color-management reporting or preservation.
- No delegates, MagickCore, or MagickWand.
- No Homebrew/core, crates.io, Windows, or hosted macOS/iOS Actions.
