# IMX v0.8.0 PNG Raster Slice

Status: published contract for the v0.8.x Linux release and tap distribution.
v0.8.x support is claimed only for archive targets present in the published
GitHub `SHA256SUMS` and the tap formula generated from those checksums.

## Objective

Make IMX materially more usable by adding the first mainstream raster format
without turning the project into a full ImageMagick clone. v0.8.0 adds a
bounded, Rust-native PNG codec and keeps the existing `imx` CLI contract.

## Supported Delta

- `imx identify input.png` and `imx identify PNG:input.png`.
- `imx input.png output.{ff,qoi,pbm,pgm,png,ppm}`.
- `imx input.{ff,qoi,pbm,pgm,ppm} output.png`.
- Exact uppercase `PNG:` prefix behavior matches the existing prefix contract.
- PNG input is detected by exact signature before extension fallback.
- Static non-interlaced grayscale, RGB, RGBA, and grayscale-alpha PNG inputs
  are supported for 8-bit and 16-bit samples.
- Grayscale-alpha PNG input normalizes to RGBA in the IMX image model.
- PNG output is deterministic from decoded pixels and does not preserve source
  compression, filter choices, ancillary chunks, profiles, gamma, comments,
  timestamps, EXIF, or other metadata.

## Preserved Contract

- Exact uppercase prefixes remain path adapters only; they confirm selected
  input/output format and do not add extensionless output selection.
- Same-format rewrites still require different paths and remain deterministic
  decode/re-encode operations.
- The product binary remains `imx`, not `magick`.
- ImageMagick remains a test/benchmark oracle only and is not linked by shipped
  binaries.

## Non-Goals

- No APNG, indexed/palette PNG, low-bit 1/2/4 sample PNG, or `tRNS` color-key
  transparency.
- No PNG color management, ICC/gamma/sRGB semantics, metadata preservation, or
  byte-preserving PNG rewrites.
- No JPEG, TIFF, GIF, WebP, PAM, PFM, BMP, stdin/stdout, `magick` alias, full
  ImageMagick CLI parser, delegates, profiles, transforms, MagickCore, or
  MagickWand.
- No crates.io, Homebrew/core, Windows, or hosted macOS/iOS release proof in
  this slice.
- No hosted macOS or iOS GitHub Actions.

## Required Evidence

The release is ready to claim only after these gates pass from the committed
source state:

```sh
cargo fmt --all -- --check
cargo fmt --manifest-path fuzz/Cargo.toml --all -- --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
IMAGEMAGICK_MAGICK=target/local-tools/magick-oracle IMX_REQUIRE_ORACLE=1 cargo test --test differential -- --nocapture
IMAGEMAGICK_MAGICK=target/local-tools/magick-oracle bash scripts/differential-corpus.sh
IMX_FUZZ_MAX_TOTAL_TIME=5 bash scripts/run-fuzz.sh
IMAGEMAGICK_MAGICK=target/local-tools/magick-oracle IMX_BENCH_ITERATIONS=3 bash scripts/bench-release.sh
IMAGEMAGICK_MAGICK=target/local-tools/magick-oracle IMX_BENCH_BASE_REF=v0.5.0 IMX_BENCH_ITERATIONS=3 bash scripts/bench-regression.sh
IMX_INSTALL_REPO_URL=/Users/jk/Desktop/imx IMX_INSTALL_REVISION=HEAD bash scripts/verify-install.sh
bash scripts/package-release.sh
```

Archive smoke should be run against published release assets before publication
claims are made:

```sh
IMX_VERSION=v0.8.0 IMX_RELEASE_TARGET=x86_64-unknown-linux-gnu \
  bash scripts/verify-release-archive.sh
```

## Tap Gate

The `jskoiz/homebrew-imx` tap must be updated from the published v0.8.0
`SHA256SUMS`, not from local archives. Tap support is claimed only after the
formula URL/SHA blocks match published release assets and Linux-only tap smoke
passes.
