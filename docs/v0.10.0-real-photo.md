# IMX v0.10.0 JPEG EXIF Orientation Fixture Reliability Slice

v0.10.0 keeps the v0.9.0 JPEG surface and adds bounded read-only EXIF
Orientation handling for synthetic JPEG photo-style fixtures. It does not claim
coverage from an externally sourced real-photo corpus.

## Contract

- `.jpg`, `.jpeg`, and exact `JPEG:` remain the only JPEG path/prefix forms.
- JPEG identify/decode stays limited to 8-bit grayscale and RGB streams.
- EXIF Orientation values 1 through 8 are read before decode.
- Decoded pixels are normalized according to the Orientation value.
- `identify` reports oriented dimensions for JPEGs with Orientation 5 through 8.
- Other EXIF, ICC, XMP, density, thumbnail, timestamp, GPS, camera, profile, and
  color-management metadata is not preserved or interpreted.
- JPEG output still writes deterministic quality-90 8-bit JPEG and does not
  write EXIF Orientation metadata.

## Proof Requirements

- Codec tests cover all Orientation values 1 through 8.
- CLI tests prove oriented identify/transcode dimensions and malformed EXIF
  failures through the `imx` binary.
- Malformed tests cover invalid Orientation values and malformed EXIF TIFF
  headers/offsets.
- Generated fixtures include `photo-orientation-o1.jpg` through
  `photo-orientation-o8.jpg`.
- Differential corpus compares oriented JPEG output against ImageMagick
  `-auto-orient` with the existing JPEG lossy metric recorder.
- Install, package, archive, and tap formula smoke checks include an Orientation
  6 JPEG.

## Still Out Of Scope

Progressive JPEG, CMYK/YCCK JPEG, 12-bit JPEG, arithmetic-coded JPEG, lossless
JPEG/JPEG-LS, JPEG 2000, JPEG XL, metadata preservation, EXIF writing, GPS
handling, ICC/profile transforms, color management, resize/crop/filter
transforms, full ImageMagick CLI parsing, MagickCore, and MagickWand remain out
of scope.
