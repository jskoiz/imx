# IMX v0.12.0 GLIBC Baseline Proof

Status: post-release closure evidence for the published Linux v0.12.0 release
archives.

Oracle flagged the precise glibc baseline as the only material v0.12.0 closure
gap: public docs and installer/tap caveats said the Linux archives require glibc
2.34 or newer, but the release evidence did not explicitly prove the maximum
referenced `GLIBC_*` symbol version.

## Published Assets

Release: <https://github.com/jskoiz/imx/releases/tag/v0.12.0>

- `imx-preview-0.12.0-x86_64-unknown-linux-gnu.tar.gz`
  - sha256: `7f573ef9406178651fee4c9d6603d674a8d4a41e0ef90f83742ee5b1d339acd3`
- `imx-preview-0.12.0-aarch64-unknown-linux-gnu.tar.gz`
  - sha256: `fe4cd4747a7c4ffdd09659efdbeb48dc017b6151ccf6ac59553cbb2e55279d09`

## Verification

The published archives were downloaded from the GitHub release, checked against
the release `SHA256SUMS`, extracted, and inspected with
`scripts/check-glibc-symbols.sh`.

```sh
scripts/check-glibc-symbols.sh \
  target/v0.12.0-published-assets/extracted/imx-preview-0.12.0-x86_64-unknown-linux-gnu/imx \
  target/v0.12.0-published-assets/extracted/imx-preview-0.12.0-aarch64-unknown-linux-gnu/imx
```

Result:

```text
target/v0.12.0-published-assets/extracted/imx-preview-0.12.0-x86_64-unknown-linux-gnu/imx: max GLIBC_2.34; allowed GLIBC_2.34
observed GLIBC versions: GLIBC_2.2.5, GLIBC_2.3, GLIBC_2.3.4, GLIBC_2.14, GLIBC_2.16, GLIBC_2.17, GLIBC_2.18, GLIBC_2.28, GLIBC_2.30, GLIBC_2.32, GLIBC_2.33, GLIBC_2.34
target/v0.12.0-published-assets/extracted/imx-preview-0.12.0-x86_64-unknown-linux-gnu/imx: GLIBC symbol baseline passed
target/v0.12.0-published-assets/extracted/imx-preview-0.12.0-aarch64-unknown-linux-gnu/imx: max GLIBC_2.34; allowed GLIBC_2.34
observed GLIBC versions: GLIBC_2.17, GLIBC_2.18, GLIBC_2.28, GLIBC_2.30, GLIBC_2.32, GLIBC_2.33, GLIBC_2.34
target/v0.12.0-published-assets/extracted/imx-preview-0.12.0-aarch64-unknown-linux-gnu/imx: GLIBC symbol baseline passed
```

The checker also fails when the allowed floor is lowered to `GLIBC_2.33`,
proving that the gate rejects binaries above the configured baseline.

## Conclusion

The published v0.12.0 Linux x86_64 and Linux arm64 binaries do not reference
`GLIBC_*` symbols newer than `GLIBC_2.34`. The existing installer and tap caveat
that these archives require glibc 2.34 or newer is therefore backed by current
release-asset evidence.
