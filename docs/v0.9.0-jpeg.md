# IMX v0.9.0 JPEG Slice

## Contract

v0.9.0 adds a bounded common-photo surface: `.jpg` and `.jpeg` path
extensions, exact `JPEG:` prefix handling, `identify`, and two-path transcodes
between JPEG and the existing FARBFELD/QOI/PBM/PGM/PNG/PPM formats.

JPEG support is intentionally narrow:

- Decode and identify 8-bit grayscale and RGB JPEG streams.
- Encode deterministic 8-bit JPEG at fixed quality 90.
- Report `format=JPEG width=<W> height=<H> channels=GRAY|RGB depth=8`.
- Treat same-format JPEG rewrites as lossy decode/re-encode operations.
- Reject non-opaque alpha inputs for JPEG output.
- Apply a 128 MiB decoded-pixel cap before JPEG decode allocation to keep
  decoder working-memory overhead bounded.
- Use ImageMagick only as an oracle in tests and benchmarks.

## Unsupported

IMX v0.9.0 does not support `JPG:` prefixes, progressive JPEG, CMYK/YCCK JPEG,
12-bit JPEG, arithmetic-coded JPEG, lossless JPEG/JPEG-LS, JPEG 2000, JPEG XL,
metadata preservation, EXIF, ICC, XMP, orientation handling, density,
thumbnails, color management, delegates, MagickCore, MagickWand, stdin/stdout,
or full ImageMagick CLI parsing.

## Evidence

The v0.9.0 release path must prove:

- Focused JPEG codec tests for grayscale/RGB encode/decode and CMYK rejection.
- CLI tests for `.jpg`, `.jpeg`, `JPEG:`, magic-before-extension detection,
  transcode, same-format rewrite, malformed input, prefix mismatch, same-path
  rejection, and non-opaque alpha rejection.
- Malformed and fuzz-smoke coverage for short inputs, SOI-only streams,
  truncated valid JPEG, bad marker lengths, CMYK JPEG, and random bytes.
- Differential tests and corpus cases that compare JPEG-involved transcodes with
  ImageMagick via decoded RGB8 metrics, not byte equality.
- Benchmarks for JPEG decode, encode, JPEG-to-FARBFELD, and JPEG output paths.
- Source install, package/archive smoke, conformance generation, no ImageMagick
  runtime linkage checks, and tap proof from published `SHA256SUMS`.

## JPEG Tolerance Policy

Lossy oracle evidence decodes both IMX and ImageMagick JPEG outputs to raw RGB8
and records max absolute difference, MAE, RMSE, PSNR, p99 absolute difference,
and threshold counts. The corpus keeps lossless formats byte-exact after oracle
decode and applies metric gates only to JPEG-involved cases.
