# IMX v0.14.0 Resize-Fit Slice

Status: v0.14.0 release contract. Release support is claimed only for archive
targets present in the published GitHub `SHA256SUMS` and tap formula generated
from those checksums, after Linux-only release/archive smoke and tap smoke pass.

## Supported Surface

- Command:
  `imx resize-fit <width>x<height> [FORMAT:]<input> [FORMAT:]<output>`.
- Formats: FARBFELD, JPEG, QOI, PBM, PGM, PNG, and PPM only.
- Prefixes: exact uppercase `FARBFELD:`, `JPEG:`, `QOI:`, `PBM:`, `PGM:`,
  `PNG:`, and `PPM:` keep the existing confirm-only behavior.
- Dimensions: lowercase `<width>x<height>` with non-zero unsigned 32-bit
  decimal values.

## Behavior Contract

Resize-fit decodes the input image, computes the largest integer output size
that fits within the requested box while preserving source aspect ratio, then
runs the same center-sampled nearest-neighbor scaling used by `imx resize`.

Given source `sw` by `sh` and requested box `bw` by `bh`:

```text
if bw * sh <= bh * sw:
  output_width = bw
  output_height = max(1, round_half_up(sh * bw / sw))
else:
  output_width = max(1, round_half_up(sw * bh / sh))
  output_height = bh
```

The scaled coordinate for each destination axis is unchanged from exact resize:

```text
floor(((2 * dst + 1) * src_size) / (2 * dst_size))
```

Resize-fit does not change exact `imx resize` behavior. It does not interpolate,
crop, fill, composite alpha, preserve metadata, perform color management, or
choose a new bit depth. Existing encoder rules still decide JPEG loss,
QOI/Netpbm quantization, PBM thresholding, alpha rejection, and metadata loss.

## Evidence Requirements

- Core image tests for ImageMagick-matched fit dimensions, format preservation,
  invalid dimensions, oversized output rejection, and center-sampled pixels.
- CLI tests for resize-fit success across all seven formats, exact prefixes,
  invalid dimensions, unknown/missing/mismatched prefixes, extensionless output
  rejection, and same-path rejection.
- ImageMagick oracle proof against `-filter Point -resize <width>x<height>`
  without `!`, with exact decoded pixels for non-JPEG formats and existing JPEG
  tolerance metrics for JPEG output.
- Differential corpus coverage for plain and prefixed resize-fit across all
  seven formats.
- Install, package, release-archive, conformance, and Homebrew formula/archive
  smoke checks that exercise resize-fit.

## Non-Goals

No changes to `imx resize`, new formats, stdin/stdout, batch mode, crop, rotate,
fill, percentages, `WIDTHx`, `xHEIGHT`, `>`, `<`, `^`, filters beyond nearest
neighbor, metadata preservation, color management, `magick` alias, full
ImageMagick CLI parsing, delegates, MagickCore, MagickWand, Homebrew/core,
crates.io, Windows, or hosted macOS/iOS GitHub Actions.
